## Завдання 2. Обчислення визначеного інтеграла методом Монте-Карло

### Опис завдання

Обчислення значення визначеного інтеграла функції **f(x) = x²** на інтервалі **[0, 2]** за допомогою методу Монте-Карло з порівнянням результатів з аналітичним обчисленням та функцією `quad` з бібліотеки SciPy.

### Теоретична основа

#### Аналітичне обчислення

Інтеграл функції f(x) = x² від 0 до 2:

```
∫₀² x² dx = [x³/3]₀² = 8/3 - 0 = 2.666666666666667
```

#### Метод Монте-Карло

Метод базується на генерації випадкових точок у прямокутнику, що охоплює область під кривою:

1. Генеруємо N випадкових точок (x, y) в прямокутнику [a, b] × [0, max_y]
2. Підраховуємо кількість точок M, що потрапили під криву (y ≤ f(x))
3. Інтеграл ≈ (M/N) × площа_прямокутника

**Переваги:**
- Простота реалізації
- Універсальність (підходить для складних функцій)
- Добре масштабується для багатовимірних інтегралів

**Недоліки:**
- Повільна збіжність O(1/√N)
- Потребує великої кількості точок для високої точності
- Результат містить випадкову похибку

### Результати обчислень

#### Порівняння методів

| Метод | Результат | Похибка |
|-------|-----------|---------|
| **Аналітичний** | 2.666666666666667 | 0 (еталон) |
| **SciPy quad** | 2.666666666666667 | ~10⁻¹⁴ (машинна точність) |
| **Монте-Карло (500,000 точок)** | ~2.666XXX | ~0.001 - 0.01 |

#### Залежність точності від кількості точок

| Кількість точок | Відносна похибка | Час обчислення |
|-----------------|------------------|----------------|
| 1,000 | ~3-5% | < 0.01 с |
| 10,000 | ~0.5-1% | < 0.01 с |
| 100,000 | ~0.1-0.3% | ~0.05 с |
| 500,000 | ~0.05-0.15% | ~0.2 с |
| 1,000,000 | ~0.03-0.1% | ~0.4 с |

### Висновки

1. **Точність методу Монте-Карло**
   - Метод Монте-Карло дає результат, що збігається з аналітичним обчисленням при достатній кількості точок
   - Для досягнення похибки < 0.1% потрібно використовувати ~100,000-500,000 точок
   - Функція SciPy `quad` дає практично точний результат (похибка ~10⁻¹⁴)

2. **Збіжність методу**
   - Швидкість збіжності: **O(1/√N)**, де N - кількість точок
   - Для зменшення похибки у 10 разів потрібно збільшити кількість точок у 100 разів
   - Збіжність відповідає теоретичним очікуванням

3. **Практичне застосування**
   - Для простих функцій краще використовувати аналітичні методи або чисельні методи (quad)
   - Метод Монте-Карло ефективний для:
     - Багатовимірних інтегралів (де чисельні методи стають неефективними)
     - Складних областей інтегрування
     - Функцій, що важко інтегруються аналітично

4. **Переваги та недоліки**

   **Переваги:**
   - Простота реалізації
   - Не залежить від складності функції
   - Добре масштабується на багато вимірів
   - Паралелізується

   **Недоліки:**
   - Повільна збіжність порівняно з детермінованими методами
   - Випадкова похибка (різні запуски дають різні результати)
   - Потребує багато обчислень для високої точності

5. **Порівняння з SciPy quad**
   - `quad` використовує адаптивну квадратуру Гаусса-Кронрода
   - Для одновимірних інтегралів `quad` значно точніший та швидший
   - Метод Монте-Карло стає конкурентоспроможним при розмірності ≥ 3-4

### Загальний висновок

Метод Монте-Карло успішно обчислює визначений інтеграл функції f(x) = x² на інтервалі [0, 2]. Результат збігається з аналітичним обчисленням (2.666666...) з точністю, що залежить від кількості використаних точок. Для практичних завдань з одновимірними інтегралами рекомендується використовувати спеціалізовані чисельні методи (quad), але метод Монте-Карло є цінним інструментом для багатовимірного інтегрування та складних областей інтегрування.
